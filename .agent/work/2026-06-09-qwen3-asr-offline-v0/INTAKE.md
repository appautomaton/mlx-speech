## Objective
Add a bounded Qwen3-ASR offline v0 spec for the "I speak, it becomes text" pipeline, with primary support for English, Chinese, and mixed Chinese/English transcription while fitting the existing `mlx-speech` ASR interface and pure-MLX runtime rules.

## Broader Intent
Support Qwen3-ASR as a strong English/Chinese ASR family in this project without breaking existing Cohere ASR or Granite Speech ASR behavior.

## What Makes This Cool
Qwen3-ASR is attractive because it can transcribe natural code-switched speech where Chinese and English are mixed in one utterance. The first useful version should optimize for English, Chinese, and Chinese/English mixed speech, and make local speech-to-text feel native to the library rather than a wrapper around the upstream PyTorch or vLLM implementation.

## Closure Target
Reference system: `.references/Qwen3-ASR` at commit `c17a131fe028b2e428b6e80a33d30bb4fa57b8df`, recorded in `docs/references.md`.

Closed means the spec defines an MLX-native offline transcription path that maps the reference pipeline into local project boundaries: audio normalization, WhisperFeatureExtractor-parity feature extraction, prompt construction, audio placeholder expansion, audio encoder, Qwen3 text decoder, audio-text fusion, greedy decoding, output parsing, adapter registration, and verification.

## Constraints
- Runtime inference must stay pure MLX.
- Do not add `torch`, `torchaudio`, `transformers`, `vLLM`, or the upstream `qwen-asr` package as runtime dependencies.
- Model weights remain local and out of git.
- Apple Silicon can run 16-bit inference efficiently; v0 should preserve the upstream BF16 precision unless a later optimization pass proves quantization is needed.
- `.references/Qwen3-ASR` is read-only source evidence, not vendored runtime code.
- The public ASR surface must remain clean and avoid Qwen-specific leakage unless required for mixed-language behavior.

## Scope Coverage
- Included: Qwen3-ASR as a new ASR model family; offline local-path single-pass transcription with preflight input limits; 16 kHz audio handling; English, Chinese, and mixed Chinese/English output; processor, audio encoder, text decoder, fusion, greedy generation, parser, adapter, registry, docs, and tests.
- Deferred: streaming because the reference implementation routes it through vLLM; timestamps because Qwen forced alignment uses a separate model; long-audio energy-boundary chunking and chunk merge; automatic model download because this project is local-path-first for model ports; broader multilingual validation beyond English/Chinese; int8/low-bit quantization and performance tuning until BF16 correctness is proven.
- Anti-goals: no torch-backed runtime under an MLX label; no vendoring upstream Qwen source; no changes that alter Cohere ASR or Granite Speech ASR behavior.
- Resolved decision: Qwen3-ASR v0 defaults to English/Chinese auto or mixed-language transcription; `None`, empty, or `"auto"` means auto mode; explicit Qwen options may force a language, but no silent English default is applied through either Python API or CLI.

## Scope Preservation
This preserves the user's full stated intent for an offline "speech to text" Qwen3-ASR v0 while intentionally deferring streaming, timestamps, and distribution conveniences that are separate capabilities.

## Recommended Approach
Offline Qwen3-ASR v0: a complete but bounded spec for local checkpoint inference, excluding streaming, timestamps, automatic downloads, and optimization work.

## Key Assumptions and Risks
- The reference pipeline can be reproduced in MLX from local checkpoint files and tokenizer assets.
- The largest implementation risks are Qwen3 text decoder parity, KV cache/RoPE behavior, audio placeholder length matching, and checkpoint key remapping.
- The current ASR protocol default `language="en"` conflicts with Qwen's desired auto/mixed-language behavior; this change resolves it by allowing `language=None`/`"auto"` semantics while preserving Cohere and Granite default behavior internally.
- Exact audio feature extraction parity and audio feature length math are high-risk enough to plan as their own slice.
- The downloaded upstream checkpoint is already BF16 safetensors; any v0 conversion should be key/layout sanitization or MLX packaging, not precision reduction.

## Rejected or Deferred

- Broad ASR architecture-first work was deferred because it would expand the blast radius beyond Qwen3-ASR v0.
