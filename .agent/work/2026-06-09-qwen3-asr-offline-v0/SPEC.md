## Bounded Goal
Add Qwen3-ASR as a new MLX-native offline ASR family that turns local speech audio into text, with primary support for English, Chinese, and mixed Chinese/English transcription by default.

## Broader Intent
Make Qwen3-ASR available through the existing `mlx-speech` ASR interface without breaking Cohere ASR, Granite Speech ASR, or the project's pure-MLX runtime boundary.

## Work Classification
- Work scale: capability
- Work shape: mixed feature/parity/coverage
- Selected lenses: product, engineering, runtime

## Target Stakeholder
Library users and maintainers who want local Apple Silicon speech-to-text inference for English, Chinese, and Chinese/English mixed speech through the same ASR entry point used by existing model families.

## Source Evidence
- Current ASR family dispatch lives in `src/mlx_speech/asr/_registry.py` and recognizes `cohere_asr` and `granite_speech`.
- Current ASR protocol and output type live in `src/mlx_speech/asr/_adapter.py`.
- Qwen3-ASR reference source is recorded in `docs/references.md` as `.references/Qwen3-ASR` at commit `c17a131fe028b2e428b6e80a33d30bb4fa57b8df`.
- Reference inference centers on `Qwen3ASRModel.transcribe`, 16 kHz audio normalization, optional long-audio chunking, WhisperFeatureExtractor-style processor features, prompt construction, audio placeholder expansion, audio encoder features, Qwen3 text decoding, and parsed language/text output.

## Required Outcome

### R1: Family Integration
Qwen3-ASR is added as an isolated ASR family selected from `config.json` with `model_type == "qwen3_asr"`. The adapter returns `ASROutput(text, language)` and does not alter Cohere ASR or Granite Speech ASR behavior.

### R2: Local Checkpoint Boundary
Runtime loading is local-path-first and uses explicit config parsing plus explicit weight remapping. The upstream checkpoint is already BF16 `.safetensors`; v0 targets an MLX-loadable BF16 checkpoint path. Any conversion is for key/layout sanitization and packaging, not precision reduction. Weights are not committed. Conversion/loading code stays separate from inference code.

### R3: Audio Frontend
The v0 runtime accepts audio paths, NumPy arrays, and MLX arrays, normalizes to 16 kHz mono, and implements WhisperFeatureExtractor-parity log-mel features for Qwen3-ASR: 128 mel bins, `n_fft=400`, `hop_length=160`, log10 normalization with dynamic-range compression, dynamic/longest padding rather than Whisper's fixed 30-second padding, and exact audio token length calculation without adding torch, torchaudio, transformers, vLLM, or upstream `qwen-asr` as runtime dependencies.

### R4: Prompt And Language Semantics
The prompt builder follows the reference chat-template intent and expands audio placeholders so placeholder count matches computed audio feature lengths. Qwen3-ASR defaults to auto English/Chinese/mixed-language transcription: `None`, empty string, and `"auto"` map to reference auto mode with no forced language suffix. Explicit supported languages map to forced text-only transcription. The Qwen adapter uses `language: str | None = None`, the shared ASR protocol allows `None`, Cohere and Granite map `None` to their current English default internally, and the CLI default does not pass a forced English value.

### R5: MLX Runtime Model
The runtime implements the Qwen3-ASR offline single-pass path in MLX: audio encoder, Qwen3-compatible text decoder, audio-text embedding fusion, context/window preflight, greedy decode, and output parsing. Long-audio energy-boundary splitting and chunk merge are deferred.

### R6: Verification Coverage
The implementation includes focused tests for config parsing, registry dispatch, processor length math, placeholder expansion, output parsing, checkpoint key mapping, and family isolation. End-to-end waveform transcription is validated only with an available local checkpoint.

### R7: Documentation
Docs explain supported v0 behavior, local checkpoint expectations, English/Chinese/mixed-language defaults, deferred features, and the reference commit used for implementation.

## Scope Coverage Decisions
- Included: offline local Qwen3-ASR single-pass transcription with preflight input limits; English, Chinese, and mixed Chinese/English default behavior; processor, audio frontend, model runtime, adapter, registry, docs, tests, and checkpoint mapping.
- Deferred: streaming, timestamps, forced aligner, long-audio energy-boundary splitting and chunk merge, automatic model download, broader multilingual validation beyond English/Chinese, int8/low-bit quantization, and performance tuning beyond BF16 correctness.
- Anti-goals: no torch-backed runtime under an MLX label; no vendored upstream Qwen source; no changes to Cohere ASR or Granite Speech ASR behavior.
- Resolved decision: Qwen3-ASR v0 uses auto English/Chinese/mixed language by default and only forces language through an explicit option. The default auto mode is reachable from both the Python API and CLI.

## Constraints And Risks
- Pure MLX runtime is mandatory.
- Do not add `torch`, `torchaudio`, `transformers`, `vLLM`, or upstream `qwen-asr` as runtime dependencies.
- The reference repo can guide behavior but cannot become runtime code.
- Apple Silicon BF16 inference is an explicit v0 target; low-bit quantization is not required before proving correctness.
- The largest engineering risks are WhisperFeatureExtractor parity, Qwen3 decoder parity, RoPE/KV cache behavior, audio placeholder length alignment, and weight key remapping.
- Full end-to-end verification requires a local converted checkpoint; without one, implementation can verify non-checkpoint tests but cannot claim completed speech-to-text support.

## Acceptance Criteria
- AC1: `model_type == "qwen3_asr"` resolves to a Qwen ASR adapter while `cohere_asr` and `granite_speech` still resolve as before.
- AC2: Qwen config parsing covers thinker/audio/text config fields and audio token IDs required by the reference pipeline.
- AC3: Audio frontend tests verify 16 kHz handling, WhisperFeatureExtractor-parity feature shape, dynamic padding, attention mask behavior, and audio output length math without checkpoint files.
- AC4: Prompt tests verify chat prompt construction and audio placeholder expansion count for one or more audio inputs.
- AC5: Output parser tests cover detected language output, forced-language output, empty language output, no-language-tag output, and English/Chinese mixed text output.
- AC6: Checkpoint tests verify expected Qwen key remapping and skip cleanly when local weights are absent.
- AC6a: Checkpoint packaging preserves BF16 tensors unless a tensor must change layout for MLX correctness.
- AC7: Runtime code has a preflight guard for context/window size before generation.
- AC8: With a local converted checkpoint, a runtime smoke test transcribes a short single-pass audio sample to non-empty text.
- AC9: With suitable local English, Chinese, and mixed Chinese/English samples, default Qwen transcription does not force English and preserves both Chinese and English text when the model output supports it; this default is reachable through both Python API and CLI without passing a language.
- AC10: Project dependency metadata does not add forbidden runtime dependencies.
- AC11: Omitting language for Cohere ASR or Granite Speech ASR preserves their current default behavior.
- AC12: `pytest tests/unit/` passes before reporting implementation work complete.

## Anti-Goals
- Streaming ASR support.
- Timestamp or word-level alignment support.
- Forced-aligner model support.
- Automatic checkpoint download or hub alias support before a converted MLX checkpoint convention exists.
- Guaranteeing or validating broader multilingual behavior beyond English and Chinese in v0.
- Long-audio energy-boundary splitting and chunk merge.
- Int8 or other low-bit quantization in v0.
- Torch, transformers, vLLM, or upstream-package backed inference.
- Broad public ASR API redesign beyond the minimal `language=None` / `"auto"` semantics needed to make Qwen auto mode reachable.
