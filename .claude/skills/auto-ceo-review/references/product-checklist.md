# Product Review Checklist

Run these checks before rendering a verdict. They are product-focused, not implementation-focused.

## Premise Challenge

1. **Is this the right problem to solve?** Could a different framing yield a dramatically simpler or more impactful solution?
2. **What is the actual user/business outcome?** Is the plan the most direct path, or is it solving a proxy problem?
3. **What would happen if we did nothing?** Real pain point or hypothetical one?

## Differentiation Scan

- What is defensible or differentiated here? Why would a user choose this over doing nothing or using a competitor?
- Is the differentiation sustainable (moat) or temporary (feature parity)?
- Are we building a vitamin (nice to have) or a painkiller (must have)?

## Scope Calibration

- Is the scope narrow enough to ship and learn from?
- Can the core value be delivered in a single slice?
- What is the minimum version that proves or disproves the bet?

## Leverage Assessment

- Is this the highest-leverage use of effort right now?
- Does this unlock future capabilities, or is it a one-off?
- Are we solving a problem that will recur, or a one-time edge case?

## User Sovereignty

- Can you name the specific person whose career gets better if this ships?
- Can you name the specific person whose career gets worse if it doesn't?
- If you can't name either, you don't know who you're building for.

## Anti-Goal Filter

- What is the weakest part of the current direction?
- What is generic, low-leverage, or mis-scoped?
- What should be explicitly de-scoped?

## Temporal Check

- Does this plan move toward the 12-month ideal state or away from it?
- If this ships today, what does it make harder or easier next quarter?
