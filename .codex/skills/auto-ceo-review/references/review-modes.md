# Review Modes

Select one mode at the start of the review and commit to it. Do not drift.

## SCOPE EXPANSION

You are building a cathedral. Envision the platonic ideal. Push scope UP.

- **10x check:** What version delivers 10x more value for 2x the effort?
- **Platonic ideal:** If the best engineer had unlimited time, what would the user feel?
- **Delight opportunities:** List at least 5 adjacent improvements that make the feature sing.
- **Opt-in ceremony:** Present each expansion as a separate decision. Accepted items become plan scope. Rejected items go to "NOT in scope."

## SELECTIVE EXPANSION

Hold the current scope as baseline. Then surface expansion opportunities individually.

- Run the HOLD SCOPE analysis first.
- Then scan for: 10x check, delight opportunities, platform potential.
- **Cherry-pick ceremony:** Present each expansion individually. Neutral posture: state effort and risk, let the user decide.

## HOLD SCOPE

The plan's scope is accepted. Your job is to make the product case bulletproof.

- Complexity check: >8 files or >2 new concepts is a smell. Can the same goal be achieved more simply?
- Flag work that could be deferred without blocking the core objective.
- Do not silently reduce OR expand.

## SCOPE REDUCTION

You are a surgeon. Find the minimum viable version that achieves the core outcome.

- Ruthless cut: What is the absolute minimum that ships value? Everything else is deferred.
- Separate "must ship together" from "nice to ship together."
- Be ruthless.

## Mode Selection

| Context | Default Mode |
|---------|-------------|
| Greenfield feature | EXPANSION |
| Feature enhancement / iteration | SELECTIVE EXPANSION |
| Bug fix or hotfix | HOLD SCOPE |
| Refactor | HOLD SCOPE |
| Plan touching >15 files | REDUCTION (unless user pushes back) |
| User says "go big" / "ambitious" / "cathedral" | EXPANSION |
| User says "hold scope but tempt me" / "cherry-pick" | SELECTIVE EXPANSION |
